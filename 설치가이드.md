# 팀 레퍼런스 보드 — 설치 가이드

## 준비물
- [Node.js](https://nodejs.org) (LTS 버전)
- [Supabase](https://supabase.com) 계정 (무료)

---

## 1단계 — Supabase 프로젝트 만들기

1. https://supabase.com 접속 → **Start your project**
2. GitHub으로 로그인
3. **New project** 클릭
4. 프로젝트 이름: `refboard` (아무거나 OK)
5. DB 비밀번호 설정 (잘 저장해두세요)
6. 리전: **Northeast Asia (Seoul)**
7. **Create new project** 클릭 → 약 1분 대기

---

## 2단계 — Supabase DB 테이블 만들기

1. 왼쪽 메뉴 **SQL Editor** 클릭
2. 아래 SQL을 붙여넣고 **Run** 클릭:

```sql
-- 이미지 테이블 생성
create table images (
  id uuid default gen_random_uuid() primary key,
  name text not null,
  url text not null,
  uploader text not null,
  tags text[] default '{}',
  created_at timestamptz default now()
);

-- 누구나 읽기/쓰기 가능하게 설정 (공개 보드)
alter table images enable row level security;

create policy "누구나 조회 가능" on images
  for select using (true);

create policy "누구나 추가 가능" on images
  for insert with check (true);

create policy "누구나 삭제 가능" on images
  for delete using (true);
```

---

## 3단계 — Supabase 스토리지 버킷 만들기

1. 왼쪽 메뉴 **Storage** 클릭
2. **New bucket** 클릭
3. 이름: `images`
4. **Public bucket** 체크 ✅
5. **Create bucket** 클릭

버킷 정책 설정:
1. 버킷 선택 → **Policies** 탭
2. **New policy** → **For full customization**
3. 정책 이름: `public-access`
4. Allowed operations: **INSERT, SELECT** 모두 체크
5. Target roles: `anon` 선택
6. **Review** → **Save policy**

---

## 4단계 — API 키 복사하기

1. 왼쪽 메뉴 **Settings** → **API**
2. 두 값을 복사해두세요:
   - **Project URL** (https://xxx.supabase.co)
   - **anon public** key

NEXT_PUBLIC_SUPABASE_URL= https://yixibosrtrtdczdzheqf.supabase.co

## 5단계 — 프로젝트 설정하기

터미널(Mac: Terminal, Windows: PowerShell)에서:

```bash
# 프로젝트 폴더로 이동
cd refboard

# 환경변수 파일 만들기
cp .env.local.example .env.local
```

`.env.local` 파일을 텍스트 에디터로 열고 복사한 값 붙여넣기:
```
NEXT_PUBLIC_SUPABASE_URL=https://xxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGci...
```

---

## 6단계 — 실행하기

```bash
# 패키지 설치
npm install

# 개발 서버 시작
npm run dev
```

브라우저에서 http://localhost:3000 접속하면 끝!

---

## 7단계 — 팀원과 공유하기

로컬에서 잘 되면 Vercel로 배포하면 팀 전체가 쓸 수 있어요:

1. https://vercel.com 접속 → GitHub 로그인
2. **New Project** → GitHub 저장소 연결
3. **Environment Variables** 에 `.env.local` 내용 입력
4. **Deploy** 클릭

배포 완료되면 `https://xxx.vercel.app` 링크가 생겨요. 그 링크를 팀원에게 공유하면 끝!

## 문제 해결

| 문제 | 해결 |
|------|------|
| `npm install` 오류 | Node.js 버전 확인 (18 이상 필요) |
| 이미지 업로드 안 됨 | Supabase Storage 버킷이 Public인지 확인 |
| 데이터 안 나옴 | `.env.local` 값이 맞는지 확인 |
| 빌드 오류 | `npm run build` 후 오류 메시지 확인 |
